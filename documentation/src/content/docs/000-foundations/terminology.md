# 用語解説

本プロジェクトで頻出する技術用語の解説。

## DWH（Data Warehouse）

**データウェアハウス**。分析目的でデータを蓄積・統合する専用データベース。

### 特徴

| 観点 | 業務DB（OLTP） | DWH（OLAP） |
|------|---------------|-------------|
| 目的 | トランザクション処理 | 分析・レポーティング |
| データ形式 | 正規化（重複排除） | 非正規化（結合済み） |
| 更新頻度 | リアルタイム | バッチ（日次等） |
| クエリ | 単純・高頻度 | 複雑・低頻度 |

### 本プロジェクトでの適用

個人のログデータ（時間記録、健康データ等）を複数サービスから収集し、Supabase（PostgreSQL）上で4層アーキテクチャとして構築：

```
raw → staging → core → marts
```

## BI（Business Intelligence）

**ビジネスインテリジェンス**。データを可視化・分析し、意思決定を支援する技術・プロセスの総称。

### 構成要素

- **ダッシュボード**: KPIやメトリクスの可視化
- **レポート**: 定型・アドホックな分析レポート
- **OLAP**: 多次元分析（ドリルダウン、スライス&ダイス）

### 本プロジェクトでの適用

- Grafana Cloudでダッシュボード構築
- marts層のビュー・テーブルを直接参照
- 個人の「振り返り」「改善」のためのインサイト提供

## ML（Machine Learning）

**機械学習**。データからパターンを学習し、予測・分類を行うアルゴリズムの総称。

### 代表的な手法

| カテゴリ | 手法例 | 用途 |
|---------|--------|------|
| 教師あり学習 | 回帰、分類 | 予測、ラベル付け |
| 教師なし学習 | クラスタリング、次元削減 | グループ化、可視化 |
| 強化学習 | Q学習、方策勾配 | 最適行動の学習 |

### 本プロジェクトでの関連

現時点では直接的なML活用は限定的だが、marts層で整備されたデータは将来的なML分析の基盤となる：
- 時間利用パターンの予測
- 異常検知（習慣の逸脱）
- 分類（活動タイプの自動判定）

## LLM（Large Language Model）

**大規模言語モデル**。大量のテキストデータで学習した、自然言語を理解・生成するAIモデル。

### 代表例

- GPT-4、GPT-4o（OpenAI）
- Claude（Anthropic）
- Gemini（Google）

### 従来のMLとの違い

| 観点 | 従来のML | LLM |
|------|---------|-----|
| 入力 | 構造化データ | 自然言語 |
| タスク定義 | 事前設計が必要 | プロンプトで柔軟に指定 |
| 汎用性 | タスク特化 | 汎用的 |

### 本プロジェクトでの適用

**MCP（Model Context Protocol）前提の設計**：
- LLMがデータベースに直接アクセスし、自然言語で分析
- 従来のBIダッシュボードを補完・代替
- marts層のスキーマ設計でコンテキスト効率を重視

```
ユーザー → LLM → MCP → Supabase → 分析結果
```
