# 非機能要件

## 1. 可用性

| 項目 | 要件 |
|------|------|
| 稼働時間 | データベース・管理UIは常時稼働、同期処理はオンデマンド |
| 計画停止 | システム自体をユーザが所有するためなし |
| 障害復旧 | 手動復旧で可（自動復旧は不要） |

## 2. 性能

| 項目 | 要件 |
|------|------|
| 同期処理時間 | 1サービスあたり1分以内 |
| 管理画面応答 | 3秒以内 |
| 同時接続 | 管理画面：1ユーザー（オーナーのみ）、データベース：2～3ユーザ(LLM) |

## 3. セキュリティ

| 項目 | 要件 |
|------|------|
| 認証情報保護 | Authenticated Encryption with Associated Data - AEAD |
| 通信 | HTTPS必須 |
| アクセス制御 | オーナーのみアクセス可能 |
| 監査ログ | 同期実行ログを保持 |

## 4. 拡張性

| 項目 | 要件 |
|------|------|
| サービス追加 | 新規サービスの追加が容易な設計 |
| スキーマ変更 | マイグレーションによる安全な変更 |

## 5. 運用性

| 項目 | 要件 |
|------|------|
| デプロイ | Supabase / Vercel |
| 監視 | Supabaseダッシュボードで確認 |
| バックアップ | Supabaseの自動バックアップに依存 |
| ログ | 同期ログをDBに記録 |

## 6. コスト

| 項目 | 要件 |
|------|------|
| インフラ | 無料〜低コスト（Supabase Free/Pro） |
| 運用工数 | 最小限（自動化優先） |

## 7. 互換性

| 項目 | 要件 |
|------|------|
| ブラウザ | Chrome/Firefox/Safari最新版 |
| API | 各サービスのAPI仕様変更に追従 |

## 8. 保守性

| 項目 | 要件 |
|------|------|
| コード品質 | 型安全（TypeScript/Python型ヒント） |
| テスト | 主要機能のユニットテスト |
| ドキュメント | 設計ドキュメントの整備 |
