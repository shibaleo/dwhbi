# 受け入れテスト計画（Level 4）

## 方針

ユーザー要件の充足を確認する。本番環境で実ユーザー視点のテストを実施する。Playwright を使用して console の主要ユーザーフローをブラウザ上で自動テストする。リリース前に手動または定期実行する。

## 要件検証

| 要件 | テスト内容 |
|------|-----------|
| 日次同期 | 毎日 00:00 JST にデータが更新されるか |
| 日次レポート | PDF が正しく生成され、Google Drive に保存されるか |
| ダッシュボード | Grafana で最新データが表示されるか |
| 管理画面 | intent の入力・保存が正常に動作するか |
| 予測精度 | estimate と actual の乖離が許容範囲内か |
| 調整効果 | target 適用後の改善が確認できるか |

## UI シナリオ

| シナリオ | 操作内容 |
|----------|----------|
| サービス連携 | API Key 入力 → 保存 → 連携状態確認 |
| 手動同期 | 同期ボタン押下 → 完了確認 |
| 設定変更 | GitHub PAT 更新 → Actions 使用量表示 |
| エラー表示 | 無効な認証情報 → エラーメッセージ確認 |
| intent 入力 | 値入力 → 保存 → 反映確認 |

## 特徴

- 本番環境
- 実ユーザー視点
- 手動 or 定期実行
