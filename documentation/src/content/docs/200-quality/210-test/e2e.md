# E2Eテスト計画

## 方針

実際のユーザー操作をシミュレートし、UI から DB までの一貫した動作を検証する。Playwright を使用して console の主要ユーザーフローをブラウザ上で自動テストする。OAuth 認証フローは手動テストとし、それ以外の CRUD 操作・同期トリガーを自動化する。リリース前および大きな UI 変更時に実行する。

## シナリオ例

| シナリオ | 操作内容 |
|----------|----------|
| サービス連携 | API Key 入力 → 保存 → 連携状態確認 |
| 手動同期 | 同期ボタン押下 → 完了確認 |
| 設定変更 | GitHub PAT 更新 → Actions 使用量表示 |
| エラー表示 | 無効な認証情報 → エラーメッセージ確認 |
