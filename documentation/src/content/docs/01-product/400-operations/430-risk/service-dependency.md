# 外部サービス依存リスク

本システムは[113 非機能要件 - インフラコスト方針](/01-product/100-development/110-requirements/113-non-functional#インフラコスト方針)に基づき金銭的コストゼロを原則とし、無料サービスに依存する。

## リスク一覧

| リスク | 発生確率 | 影響度 | 対策 |
|--------|:--------:|:------:|------|
| 無料枠超過 | 低 | 中 | 別サービスの無料枠で代替 |
| サービス終了 | 中 | 高 | 代替サービスへの移行 |
| 無料枠縮小 | 中 | 中 | 代替サービスへの移行 |

## 代替サービス一覧

| 用途 | 現行 | 代替候補 |
|------|------|----------|
| DB | Supabase (500MB) | Neon (500MB), PlanetScale (5GB) |
| CI/CD | GitHub Actions (2,000分/月) | Azure Pipelines (public無制限), CircleCI (6,000分/月) |
| ホスティング | Vercel | Cloudflare Pages, Netlify |

## 移行時の考慮事項

### DB移行

- PostgreSQL互換（Neon）であればスキーマ・クエリの変更は最小限
- Supabase Vault → 移行先の暗号化機構への切り替えが必要
- RLSポリシーの再設定

### CI/CD移行

- ワークフロー定義の書き換え（YAML構文の差異）
- シークレット管理の移行
- cron構文は各サービスで互換性あり

### ホスティング移行

- Next.js対応は各サービス共通
- 環境変数の再設定
- ドメイン設定の移行（Vercel提供ドメインを使用している場合）
