# WBS・マイルストーン

## 方針

プロジェクト全体を Phase（v0.1.0, v0.2.0, v1.0.0）単位で分解し、各 Phase 内をサブプロジェクト（pipelines, transform, analyzer, console）ごとにタスク化する。マイルストーンは各バージョンのリリースとし、完了条件は ADR-001 に定義済み。

## プロジェクト管理ツール

**Jira** を採用。

### 採用理由

1. **アジャイルの型を学べる**: Jiraはスクラム/カンバンのUI・データ構造が型として組み込まれており、柔軟なプロジェクト管理ツール（Notion, Trelloなど）よりもアジャイル手法の正しい使い方を強制的に学べる
2. **API提供**: REST APIを通じてタスクデータを取得でき、本プロジェクトのDWHに統合することで時間管理（Toggl）と合わせた全体把握が可能になる
3. **業界標準**: エンタープライズでの採用率が高く、ポートフォリオとしてのアピールにもなる

### 将来の統合計画

Jira APIからデータを取得し、pipelines に `jira` サービスを追加予定：

```
raw.jira__issues        # 課題データ
raw.jira__sprints       # スプリントデータ
raw.jira__worklogs      # 作業ログ
```

これにより、タスク管理と時間記録（Toggl）を統合し、以下の分析が可能になる：

- タスクごとの実績時間 vs 見積もり時間
- スプリントベロシティの推移
- 作業カテゴリ別の時間配分
