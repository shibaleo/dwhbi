# ユーザーの責務最小化

本システムは、ユーザーが**行動選択に集中**できるよう設計されている。

## 設計思想

QPIモデルの4 practicesのうち、ユーザーが担うのは以下の2つのみ：

| practice | 担当 | 内容 |
|----------|------|------|
| **aim** | ユーザー | draft（願望・理想）を設定する |
| **log** | ユーザー/自動 | 実際の行動を記録する（Toggl等で自動化可能） |
| **analyze** | システム | actual → estimate を算出 |
| **adjust** | システム（LLM） | draft + estimate + 傾向 → target を生成 |

## ユーザーがやること

1. **願望を設定する**: 「来週は○○時間働きたい」という理想（draft）を入力
2. **行動を記録する**: 実際の作業をToggl等で記録（習慣化すれば自動）
3. **targetを信じて行動する**: システムが算出した現実的な目標に従う

## システムがやること

1. **analyze**: actualを集計・分析してestimateを算出
2. **adjust**: draft + estimate + ユーザー傾向 + 外部制約 → targetを生成

## 従来との比較

| 観点 | 従来のアプローチ | 本システム |
|------|------------------|-----------|
| 目標設定 | 「どれくらいが妥当か」を自分で考える | 願望だけ言えばいい |
| 現実との調整 | 経験と勘で調整、乖離で挫折 | LLMが現実的に調整 |
| 計画倒れ | 過大目標→未達→自己嫌悪 | targetは達成可能な値 |
| 分析 | 自分でデータを見て傾向を把握 | 自動でestimate算出 |

## 認知負荷の削減

- 「どれくらいが妥当か」を考えなくていい
- 過去データの分析をしなくていい
- 目標と実績の乖離を自分で解釈しなくていい

ユーザーは**願望を表明し、誠実に行動する**ことだけに集中できる。
